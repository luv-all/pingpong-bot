# AXL Linear Rail Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Windows에서 Ajinextek `AXL.dll`로 리니어 레일을 조그하고, `RealHardware::read_pose`가 실측 `rail_x`(m)를 반환하게 한다.

**Architecture:** 순수 `RailConfig`+클램프는 OS 무관으로 두고, `libloading` FFI와 `AxlRail`은 `#[cfg(all(windows, feature = "real"))]`로 격리한다. `.mot`는 쓰지 않고 TOML 상수만 `AxmMotSet*` / `AxmSignalSetSoftLimit`로 적용한다. 스윙 `command`의 레일은 이번에도 경고만 한다.

**Tech Stack:** Rust 2024, `libloading`, serde/TOML, clap, `thiserror`/`anyhow`, existing `HwError`

**Spec:** `docs/superpowers/specs/2026-07-22-axl-rail-design.md`

## Global Constraints

- Domain / planner `rail_x` 단위는 **미터**만.
- `pulse_out_method = 4`, `enc_input_method = 3` (벤치 고정 기본값).
- 오픈 시 soft-zero(`AxmStatusSetActPos/CmdPos`) **금지**.
- Soft limit: 앱 클램프 + `AxmSignalSetSoftLimit(use=1, stop_mode=0, selection=0, +x_max_m, -x_min_m)`.
- `AxmMotLoadParaAll` / `.mot` 파일 **금지**.
- `Hardware::command` 비영 레일 → `warn!`만, 팔 궤적만 실행.
- macOS / DLL 없는 환경: 매핑·클램프·TOML 단위 테스트 + `AxlRail::dry_run` / `jog-rail --dry-run`.
- `enabled=true`인데 DLL/API 실패 시 stub로 조용히 폴백하지 말고 bail.
- `libloading`은 `real` feature optional dep. FFI 심볼 로드는 **Windows + real**에서만.

**Interfaces:**
- Produces:
  - `AxlRail::dry_run(config: RailConfig) -> Result<Self, HwError>` — in-memory `position_m`, applies clamp on move, no DLL
  - `AxlRail::open(config: RailConfig) -> Result<Self, HwError>` — **Windows+real only**; other targets return `InvalidConfig`
  - `AxlRail::read_x_m(&mut self) -> Result<f64, HwError>`
  - `AxlRail::move_abs_m(&mut self, x: f64) -> Result<f64, HwError>` — returns clamped command
  - `AxlRail::move_rel_m(&mut self, dx: f64) -> Result<f64, HwError>`
  - `Drop`: live → servo off + `AxlClose` best-effort
- Consumes: `RailConfig` from Task 1

**FFI symbols to bind (stdcall):**

From `AXL.dll` (names match headers):

```text
AxlOpenNoReset(i32) -> u32
AxlClose() -> i32  // BOOL
AxmInfoIsMotionModule(*mut u32) -> u32
AxmMotSetPulseOutMethod(i32, u32) -> u32
AxmMotSetEncInputMethod(i32, u32) -> u32
AxmMotSetMoveUnitPerPulse(i32, f64, i32) -> u32
AxmMotSetMinVel(i32, f64) -> u32
AxmMotSetMaxVel(i32, f64) -> u32
AxmMotSetAbsRelMode(i32, u32) -> u32
AxmMotSetProfileMode(i32, u32) -> u32
AxmMotSetAccelUnit(i32, u32) -> u32
AxmSignalSetInpos(i32, u32) -> u32
AxmSignalSetServoAlarm(i32, u32) -> u32
AxmSignalSetLimit(i32, u32, u32, u32) -> u32
AxmSignalSetSoftLimit(i32, u32, u32, u32, f64, f64) -> u32
AxmSignalServoOn(i32, u32) -> u32
AxmStatusGetActPos(i32, *mut f64) -> u32
AxmStatusGetCmdPos(i32, *mut f64) -> u32
AxmStatusReadInMotion(i32, *mut u32) -> u32
AxmMovePos(i32, f64, f64, f64, f64) -> u32
```

`AXT_RT_SUCCESS = 0`, `STATUS_EXIST = 1` (`AXT_EXISTENCE`: NOTEXIST=0, EXIST=1), `ENABLE = 1` for servo on.

**Open sequence (live):**

1. `config.validate()`; require `enabled`
2. Load DLL from `config.dll_path`
3. `AxlOpenNoReset(irq_no)` == 0 — preserves recorded encoder/command positions
4. `AxmInfoIsMotionModule` status == exist
5. Set pulse/enc/unit/min/max/accel_unit/abs_rel/profile
6. Signal inpos/alarm/limit + soft limit from `soft_limit_args()`
7. `AxmSignalServoOn(axis, 1)`
8. Do **not** call SetActPos/SetCmdPos

**Move (live):** clamp → `AxmMotSetAbsRelMode(axis, 0)` for absolute command → `AxmMovePos(axis, x, vel, accel, decel)` → poll `AxmStatusReadInMotion` until 0 (sleep ~1 ms). Relative: `read_x_m` + dx → `move_abs_m`.

**Read (live):** `AxmStatusGetActPos`; if fail, `GetCmdPos`.
